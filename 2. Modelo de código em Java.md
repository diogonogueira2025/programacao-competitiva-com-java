Um típico template de código Java para programação competitiva se parece com isso:
```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.io.IOException;

public class Main {
    public static void main(String[] args) throws IOException {
        BufferedReader in = new BufferedReader(
            new InputStreamReader(System.in)
        );
        
        // seu código aqui
        
        in.close();
    }
}
```

Explicação:
`public class Main { ... }` declara uma classe chamada `Main` com modificador de acesso público, o que significa que ela pode ser acessada de qualquer lugar. Em Java, todo código precisa estar dentro de uma classe.

`public static void main(String[] args) throws IOException` é o ponto de entrada do programa. Cada parte da assinatura tem um motivo:
- `public` — modificador de acesso público, necessário para que a JVM consiga chamá-lo
- `static` — pertence à classe, não a uma instância; por isso a JVM consegue executá-lo sem precisar instanciar `Main`
- `void` — não retorna nenhum valor
- `main` — nome reconhecido pela JVM como ponto de entrada
- `String[] args` — vetor de Strings que recebe argumentos passados na linha de comando ao executar o `.class`; raramente utilizado em competições, mas obrigatório na assinatura
- `throws IOException` — declara que o método pode lançar exceções de I/O, necessário por causa do `BufferedReader`
