Em problemas com muitas linhas de saída, usar `System.out.println()` em loop pode ser lento — cada chamada faz um flush do buffer, acumulando overhead. A solução é construir toda a saída em um `StringBuilder` e imprimir uma única vez ao final.

Exemplo: 
Será dada uma sequência de pares de inteiros `x y`, um por linha, até o fim da entrada (EOF). Para cada par, deve-se imprimir a soma `x + y`. Por exemplo:

```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.io.IOException;

public class Main {
    public static void main(String[] args) throws IOException {
        BufferedReader in = new BufferedReader(
            new InputStreamReader(System.in)
        );
        
        StringBuilder sb = new StringBuilder();
        String linha;
        
        while ((linha = in.readLine()) != null) {
            String[] tokens = linha.trim().split(" ");
            
            int x = Integer.parseInt(tokens[0]);
            int y = Integer.parseInt(tokens[1]);
            
            sb.append(x + y).append("\n");
        }
  
        System.out.print(sb);
        in.close();
    }
}
```

Em vez de chamar `System.out.println()` a cada iteração, acumulamos tudo no `StringBuilder` com `append()` e fazemos uma única chamada ao `System.out.print()` no final. Quanto mais linhas de saída o problema exigir, maior a diferença de desempenho.
