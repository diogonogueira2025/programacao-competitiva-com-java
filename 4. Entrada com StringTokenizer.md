Em alguns problemas, a entrada pode ser uma linha com centenas de milhares de tokens. Nesses casos, usar `split()` é problemático: ele processa a linha inteira de uma vez com regex e aloca um vetor com todos os tokens na memória simultaneamente. Para entradas grandes, isso pode ser lento o suficiente para estourar o limite de tempo.

O `StringTokenizer` resolve isso lendo token por token sob demanda, sem regex e sem alocar um vetor:

```java
// entrada: "banana maçã uva abacaxi melão ..."
StringTokenizer st = new StringTokenizer(in.readLine(), " ");
while (st.hasMoreTokens()) {
    String palavra = st.nextToken();
}
```

Explicação:

O construtor `StringTokenizer(String str, String delim)` recebe dois parâmetros:
- `str` — a String a ser dividida, no caso o resultado do `in.readLine()`
- `delim` — o delimitador, ou seja, o caractere que separa os tokens. No caso `" "`, é o espaço

`hasMoreTokens()` verifica se ainda há tokens na linha, e `nextToken()` retorna o próximo. É a abordagem preferida em competições sempre que a linha de entrada for potencialmente muito grande.
